## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Records generated by AvscEventWriter from AVSC schemas currently have empty compact constructors. Two categories of schema-contract violations are silently permitted at the Java level: (1) a field declared non-nullable in the schema (no null union) accepts null, deferring the failure to serialisation time or hiding it entirely; (2) a List field accepts null elements and remains mutable after construction, violating the Avro array contract that items are non-null and the record immutability invariant.
<!-- SECTION:DESCRIPTION:END -->

## Acceptance Criteria
<!-- AC:BEGIN -->
- [ ] #1 The compact constructor of every generated top-level record includes Objects.requireNonNull(field, field) for each non-primitive field not annotated @Nullable
- [ ] #2 The compact constructor of every generated nested record includes the same requireNonNull checks for non-nullable, non-primitive fields
- [ ] #3 The compact constructor of every generated union branch wrapper record includes a requireNonNull check for its value parameter
- [ ] #4 No null checks are emitted for primitive Java types (int, long, double, float, boolean) since they cannot hold null
- [ ] #5 The null-check statements appear in all affected AVSC snapshot fixture files (expected/ directories) for existing test cases
- [ ] #6 A new test verifies that constructing a generated record with null for a non-nullable field throws NullPointerException immediately
- [ ] #7 A new test verifies that constructing a generated record with null for a @Nullable field succeeds without exception
- [ ] #8 All existing avro and avro-processor tests continue to pass without modification
- [ ] #9 For every non-nullable List<T> field, the compact constructor applies field = List.copyOf(field) immediately after the requireNonNull check
- [ ] #10 For every @Nullable List<T> field, the compact constructor applies if (field != null) { field = List.copyOf(field); } to enforce non-null elements and immutability when the list is present
- [ ] #11 A new test verifies that passing a List containing a null element throws NullPointerException at record construction time
- [ ] #12 A new test verifies that mutating the original List passed to the constructor does not affect the record's field (defensive-copy contract)
<!-- AC:END -->

### Step 1 — Add two private static predicates

```java
// Detects @Nullable (jakarta.annotation.Nullable) on a generated ParameterSpec
private static boolean isNullableField(ParameterSpec field) {
    var nullableType = ClassName.get(Nullable.class); // jakarta.annotation.Nullable, already imported
    return field.annotations().stream().anyMatch(a -> a.type().equals(nullableType));
}

// Detects List<T> using the Palantir JavaPoet rawType() accessor (confirmed in AssertionWriter)
private static boolean isListField(TypeName type) {
    return type instanceof ParameterizedTypeName p
            && p.rawType().equals(ClassName.get(List.class));
}
```

### Step 2 — Add `buildCompactConstructor` and `emitFieldValidation` helpers

Replace the three inline `MethodSpec.compactConstructorBuilder().addParameters(fields).build()`
expressions with a call to this method. It iterates the fields list once, emitting:

1. `Objects.requireNonNull(x, "x")` for each non-primitive, non-nullable field.
2. `x = List.copyOf(x)` immediately after for a non-nullable list field.
3. `if (x != null) { x = List.copyOf(x); }` for a nullable list field (no requireNonNull).

```java
private MethodSpec buildCompactConstructor(List<ParameterSpec> fields) {
    var ctor = MethodSpec.compactConstructorBuilder().addParameters(fields);
    fields.forEach(f -> emitFieldValidation(ctor, f));
    return ctor.build();
}

private void emitFieldValidation(MethodSpec.Builder ctor, ParameterSpec field) {
    var nullable = isNullableField(field);
    var list    = isListField(field.type());

    if (!nullable && !field.type().isPrimitive()) {
        ctor.addStatement("$T.requireNonNull($L, $S)", Objects.class, field.name(), field.name());
    }
    if (list) {
        if (nullable) {
            ctor.beginControlFlow("if ($L != null)", field.name())
                .addStatement("$L = $T.copyOf($L)", field.name(), List.class, field.name())
                .endControlFlow();
        } else {
            ctor.addStatement("$L = $T.copyOf($L)", field.name(), List.class, field.name());
        }
    }
}
```

Add `import java.util.Objects;` to the import block.

### Step 5 — Update `buildUnionBranchRecord`

Union branch value parameters are never nullable (the null case selects *no* branch). Apply
`requireNonNull` for reference types and `List.copyOf` when the value is a list:

```java
private TypeSpec buildUnionBranchRecord(String branchName, TypeName valueType, ClassName iface) {
    var valueParam = ParameterSpec.builder(valueType, "value").build();
    var ctor = MethodSpec.compactConstructorBuilder().addParameter(valueParam);
    if (!valueType.isPrimitive()) {
        ctor.addStatement("$T.requireNonNull(value, $S)", Objects.class, "value");
    }
    if (isListField(valueType)) {
        ctor.addStatement("value = $T.copyOf(value)", List.class);
    }
    return TypeSpec.recordBuilder(branchName)
            .addModifiers(Modifier.PUBLIC)
            .addSuperinterface(iface)
            .recordConstructor(ctor.build())
            .build();
}
```

### Step 6 — New tests in `AvscPluginTest`

All four new tests use existing `Compilation` objects where the fixture is already compiled (no
extra compilation overhead), or compile a fixture inline. They check source content — this is the
established pattern in `AvscPluginTest` and sufficient to satisfy ACs 6/7/11/12 because the
presence of `Objects.requireNonNull` / `List.copyOf` in the emitted source implies the runtime
behaviour.

**Test A — requireNonNull present for non-nullable fields (AC-1, AC-6)**
Use `nullableCompilation` (schema: id:string, name:string, description:["null","string"]).
Assert `NullableAvscEvent` source contains `requireNonNull(id, "id")`,
`requireNonNull(name, "name")`, does NOT contain `requireNonNull(description`.

**Test B — requireNonNull absent for @Nullable field (AC-7)**
Same compilation. Assert the description field has no `requireNonNull`.

**Test C — requireNonNull absent for primitives (AC-4)**
Use `simpleCompilation` (age:int, score:double, active:boolean).
Assert `SimpleAvscEvent` source does NOT contain `requireNonNull(age`,
`requireNonNull(score`, `requireNonNull(active`.

**Test D — List.copyOf emitted for array field (AC-9, AC-11, AC-12)**
Use existing array compilation (id:string, tags:array<string>).
Assert `ArrayAvscEvent` source contains `List.copyOf(tags)`.

**Test E — requireNonNull on union branch value (AC-3)**
Use `scalarunion` compilation (ExactValueString, ExactValueDouble branch wrappers).
Assert `ExactValueString` source contains `requireNonNull(value, "value")`.
Assert `ExactValueDouble` source does NOT contain `requireNonNull(value` (double is primitive).
