## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
When generating events from AVSC, allow the `@Avsc`-annotated contract interface to be declared as
`sealed` with the generated record subclasses listed in the `permits` clause.

Currently, writing:

```java
@Event(topic = "sale", serialization = Event.Serialization.AVRO)
@Avsc({"sale-created.avsc", "sale-paid.avsc"})
public sealed interface SaleEvent permits SaleCreated, SalePaid {}
```

fails to compile because `TypeMembers.permittedSubtypes()` passes error-typed `TypeMirror`s (representing
`SaleCreated` and `SalePaid` which are AP-generated types not yet compiled in round 1) to the Avro converter
writers. These writers then generate invalid Java code (e.g. using `""` as a package prefix), which causes
compilation errors.

### Root cause

`TypeElement.getPermittedSubclasses()` returns `ERROR`-kind `TypeMirror`s in annotation processing round 1
for `permits` types that are themselves generated by the AP (they don't exist yet as compiled classes).
`TypeMembers.permittedSubtypes()` did not filter these unresolved types, allowing the Avro infrastructure
writers to generate invalid converter files for them.

### Fix

Filter out `ERROR`-kind `TypeMirror`s in `TypeMembers.permittedSubtypes()`. These will be resolved correctly
in round 2 when the AP-generated records are compiled. The Avro converter writers then process the proper
`TypeManifest` objects in round 2 and generate correct converters.
