## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Generated request records (e.g. `Create{Name}Request`, `{Name}{Operation}Request`) and `@Event`-annotated records
are plain Java records today. Consuming code must use the canonical constructor, which becomes fragile as fields
are added or reordered. This task introduces a generated **builder** for every such record so that call sites can
construct instances in a readable, refactoring-safe way.

The builder is generated alongside the record, nested inside it as a `public static final class Builder`. A static
factory method `{RecordType}.builder()` returns a fresh `Builder` instance. The `Builder` exposes one fluent
setter per record component, and a terminal `build()` method that invokes the canonical constructor.

**Dependency on TASK-136**: once the Avro extraction is complete, `@Event`-annotated records that are
Avro-serialized live in `prefab-avro-processor`. The builder writer for those records must reside in
`prefab-avro-processor` as well (or be shared via a common utility in `prefab-annotation-processor`).
The implementation must not add Avro-specific builder logic into `prefab-core` or `prefab-annotation-processor`.
<!-- SECTION:DESCRIPTION:END -->
